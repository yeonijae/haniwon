# commit-push-pr

변경사항을 커밋하고, push한 후 PR을 생성합니다.

## 실행 단계

1. `git status`로 변경사항 확인
2. 변경된 파일들을 스테이징 (`git add`)
3. 커밋 메시지 작성 및 커밋
4. 원격 저장소에 push
5. GitHub PR 생성 (gh cli 사용)

## 인자

- `$ARGUMENTS`: 커밋 메시지 (선택사항). 없으면 변경사항을 분석해서 자동 생성

## 지침

변경사항을 분석하여 적절한 커밋 메시지를 작성하세요. 커밋 메시지는 한글로 작성하고, 변경 내용을 명확하게 설명해야 합니다.

PR 제목은 커밋 메시지를 기반으로 하고, PR 본문에는 변경사항 요약을 포함하세요.

현재 브랜치가 main/master가 아닌 경우에만 PR을 생성하세요. main/master 브랜치인 경우 push만 수행하세요.
