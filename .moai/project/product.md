# Haniwon (하니원) - 프로젝트 개요

| 항목 | 내용 |

## 프로젝트 설명

하니원은 한의원 업무 전반을 관리하는 통합 클리닉 관리 시스템입니다. 환자 접수부터 진료, 처방, 치료, 예약, 재고 관리까지 한의원 운영에 필요한 모든 기능을 하나의 플랫폼에서 제공합니다.

기존 EMR 시스템(오케이차트)과의 연동을 통해 환자 정보와 진료 기록을 실시간으로 조회하면서, 한의원 특화 기능들을 추가로 제공하여 업무 효율성을 극대화합니다.

## 대상 사용자

### 사용자 역할

| 역할 | 설명 | 주요 사용 모듈 |
|------|------|----------------|
| super_admin | 시스템 관리자 | 모든 모듈 접근 가능, DB 관리 |
| medical_staff | 의료진 (원장) | 원장실, 처방, 메트릭, 통계 |
| desk | 데스크 직원 | 접수/수납, 예약, 환자 관리 |
| counseling | 상담 직원 | 환자 상담, 복약 관리, CS |
| treatment | 치료실 직원 | 치료실 관리, 액팅 대기열 |
| decoction | 탕전실 직원 | 한약 관리, 재고 관리 |

### 사용 환경

- 데스크톱 웹 브라우저 (Chrome, Edge 권장)
- 태블릿 (원장 패드 전용 인터페이스)
- 내부 네트워크 환경 (192.168.0.x)

## 핵심 기능

### 1. 환자 관리 (manage)

**접수 및 대기 관리**
- 내원 환자 접수 및 대기 현황 실시간 모니터링
- 환자 정보 조회 및 검색 (EMR 연동)
- 초진/재진 구분 및 관리

**수납 처리**
- 다양한 결제 수단 지원 (현금, 카드, 계좌이체)
- 미수금 관리 및 추적
- 영수증 발행

**치료실 배정**
- 환자별 치료실 자동/수동 배정
- 치료 진행 상태 관리

### 2. 원장실 (doctor)

**차트 관리**
- 환자 진료 기록 조회 및 작성
- 진료 노트 작성 (TipTap 에디터)
- 과거 진료 이력 조회

**처방 관리**
- 한약 처방 작성
- 치료 항목 지시
- 액팅 지시 (침, 추나, 초음파 등)

**메트릭 및 통계**
- 진료 실적 대시보드
- 원장별 통계 분석
- 재진율/삼진율/이탈율 추적

### 3. 치료실 관리 (treatment)

- 치료실 현황 실시간 모니터링
- 치료 항목별 관리 (침, 뜸, 부항, 추나 등)
- 환자별 기본 치료 설정
- 치료 완료 처리

### 4. 액팅 관리 (acting)

- 실시간 액팅 대기열 관리
- 드래그앤드롭으로 순서 변경
- 액팅 유형별 분류
  - 침구: 침, 전침, 뜸
  - 추나: 추나, 교정
  - 물리치료: 초음파, 간섭파
  - 기타: 향기, 약침

### 5. 예약 관리 (reservation)

- 월간/주간/일간 캘린더 뷰
- MSSQL 연동 예약 조회
- 예약 생성, 수정, 삭제
- 예약 현황 알림

### 6. 복약 관리 (herbal)

**초진 관리**
- 당일 초진 환자 자동 감지
- 감사 메시지 발송 관리

**복약 설정**
- 고액 결제(200,000원 이상) 자동 감지
- 한약 종류별 설정 (탕약, 공진단, 경옥고)
- 복약 스케줄 관리

**콜 관리**
- 복약콜: 복용 시작 2일 후 상태 확인
- 내원콜: 복용 종료 3일 전 재진 안내
- 이벤트 혜택 안내
- 사후관리: 복용 완료 90일 후 관리

### 7. 재고 관리 (inventory)

- 약재 재고 현황 관리
- 처방 정의 및 관리
- 비품 요청 및 승인
- 재고 부족 알림

### 8. 블로그 및 콘텐츠 (blog, content)

- 블로그 콘텐츠 작성 및 관리
- Rich Text 에디터 (TipTap)
- 미디어 업로드 관리
- 페이지 조회수 통계
- 랜딩 페이지 관리

### 9. 통계 및 분석 (statistics, metrics)

- 진료 실적 대시보드
- 원장간 비교 분석
- 누적 통계 (재진율, 삼진율, 이탈율)
- 기간별 분석 리포트
- 차트 시각화 (Recharts)

### 10. 기타 기능

**포털 (portal)**
- 직원 로그인 및 인증
- 역할 기반 메뉴 접근 제어
- 세션 관리

**원장 패드 (pad)**
- 태블릿 전용 인터페이스
- 간편 진료 기록 작성
- 대기 환자 확인 및 호출

**DB 관리 (db-admin)**
- 데이터베이스 관리 도구
- SQL 쿼리 실행
- 테이블 구조 조회

**운영 매뉴얼 (wiki)**
- 시스템 사용 가이드
- 업무 프로세스 문서화

**메시징 (chat)**
- 내부 메시징 시스템
- 실시간 알림 (Socket.IO)

**검사 결과 (exam)**
- 검사 결과 조회 및 관리
- PDF 리포트 생성

**마케팅 퍼널 (funnel)**
- 마케팅 캠페인 관리
- 퍼널 분석

**직원 관리 (staff)**
- 직원 정보 관리
- 권한 설정

**고객 서비스 (cs)**
- 고객 문의 관리
- CS 이력 추적

**환자 케어 (patient-care)**
- 복약 관리 모듈로 리다이렉트

## 사용 사례

### 일반적인 하루 업무 흐름

1. **오전 업무 시작**
   - 데스크 직원이 포털에 로그인
   - 당일 예약 현황 확인
   - 초진 환자 목록 검토

2. **환자 접수**
   - 내원 환자 접수 처리
   - EMR에서 환자 정보 자동 조회
   - 대기 목록에 환자 추가

3. **진료**
   - 원장이 닥터패드에서 대기 환자 확인
   - 진료 기록 작성
   - 치료/액팅 지시

4. **치료**
   - 치료실에서 환자 수신
   - 지시된 치료 항목 수행
   - 치료 완료 처리

5. **수납**
   - 진료비 수납 처리
   - 한약 처방 시 복약 관리 설정

6. **마감**
   - 일일 통계 확인
   - 미수금 현황 점검
   - 익일 예약 확인

### 복약 관리 프로세스

1. 고액 결제 자동 감지 (200,000원 이상)
2. 복약 관리 대상자 설정
3. 자동 콜 스케줄 생성
4. 단계별 콜 진행 (복약콜 -> 내원콜 -> 사후관리)

### 통계 분석

1. 기간 설정 (일간/주간/월간)
2. 원장별 실적 비교
3. 재진율, 삼진율, 이탈율 추적
4. PDF 리포트 생성 및 다운로드
